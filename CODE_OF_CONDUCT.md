# Code of Conduct

## The standard

Treat other contributors with patience and professional respect. Give specific, evidence-based
feedback. Disagree with an idea without attacking the person behind it, and take responsibility
when you get something wrong.

Harassment, threats, sexualized attention, deliberate intimidation, insults, trolling and the
publication of somebody else's private information are not acceptable here.

ELIZA Lab deals with dialogue examples. Never post real conversations, health information,
contact details, credentials or somebody else's personal data in an issue, pull request or
discussion. Use short fictional prompts that reproduce the behavior.

## Scope and enforcement

This code applies to repository issues, pull requests, reviews and other spaces where somebody is
representing the project. Maintainers may edit or remove content, reject contributions, issue a
warning or exclude somebody from project spaces when their conduct puts people or their data at
risk.

Report a conduct concern privately to [info@ejupilabs.com](mailto:info@ejupilabs.com). For a
software vulnerability or exposed credential, use
[GitHub private vulnerability reporting](https://github.com/ejupi-djenis30/PsychologistRustBot/security/advisories/new)
instead. Reports will be handled confidentially and retaliation for a good-faith report is not
acceptable.
