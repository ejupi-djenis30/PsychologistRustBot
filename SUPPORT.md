# Support

## Something is broken

Open a [bug report](https://github.com/ejupi-djenis30/PsychologistRustBot/issues/new?template=bug_report.yml)
with the affected version, your Rust and Node.js versions, the smallest fictional reproduction,
and the checks you already ran. Do not include real conversations or personal, health, account or
contact data.

## You want to propose a change

Use the [feature request](https://github.com/ejupi-djenis30/PsychologistRustBot/issues/new?template=feature_request.yml)
and explain the learning, engine or accessibility task it would improve. The proposal must keep
the project local-only, bounded and clearly non-clinical.

## The report is sensitive

Do not open a public issue for a vulnerability or exposed credential. Follow the
[security policy](SECURITY.md) and use
[private vulnerability reporting](https://github.com/ejupi-djenis30/PsychologistRustBot/security/advisories/new).

This is an open-source educational project, not a care service. Response times are not guaranteed,
but complete reports built from fictional input are prioritized.
